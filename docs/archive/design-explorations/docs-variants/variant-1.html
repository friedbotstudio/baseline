Core concepts

Workflow phases

The eleven-phase pipeline that takes a request from intake to commit. Every phase produces a single artifact, and every transition is enforced at the write boundary by a hook.

Overview

Aurora ships a constitutional workflow with eleven ordered phases. The order is enforced at the write boundary by track_guard, so phases cannot be skipped or reordered without an explicit exception in .claude/state/workflow.json.

Three of the eleven phases are consent gates. They are user-typed slash commands that Claude is structurally unable to invoke, which keeps approval authority outside the model's reach.

The eleven phases

Each phase has exactly one mandated output. The output of phase n is the contract for phase n + 1.

#PhaseSkillOutput
1Intake/intakedocs/intake/<slug>.md
2Scout/scoutdocs/scout/<slug>.md
3Research/researchdocs/research/<slug>.md
4Spec/specdocs/specs/<slug>.md
5Approve spec/approve-specapproval token
6TDD/tddcode

8Security/securityfindings report
9Integrate/integrateverify verdict
10Document/documentprose
11Commit/commitcommit

Phase flow

Intake→ Scout→ Research→ Spec→ Approve→ TDD→ Simplify→ Integrate→ Document→ Commit

Consent gates

Each gate writes a short-lived consent marker that the corresponding PreToolUse guard validates before it permits the approval-token write. Markers expire after sixty seconds and are deleted on use, so a gate cannot be replayed.

!
Consent gates are not bypassable.

The consent_gate_grant hook runs on UserPromptSubmit, outside Claude's tool boundary. Claude cannot forge or replay an approval marker.

Customizing the pipeline

To skip a phase for a specific request, add it to the exceptions array. /triage is the only sanctioned writer of this file.

// .claude/state/workflow.json
{
  "slug": "site-react-ssg-seo",
  "entry_phase": "intake",
  "exceptions": ["security"],
  "completed": ["intake", "scout"],
  "current": "research"
}

Exceptions are surfaced in every phase log so the audit trail records which phases ran and which were waived.
